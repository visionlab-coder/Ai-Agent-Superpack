# AI 에이전트 슈퍼팩 (AI Agent SuperPack) 활용 가이드

현재 프로젝트에 설치된 모든 에이전트, 스킬, 룰, 워크플로우를 하나로 묶은 패키지입니다. `Claude Forge`의 강력한 자동화 기능과 `bkit`의 체계적인 PDCA(계획-실행-평가-개선) 방법론이 모두 통합되어 있습니다.

## 📦 패키지 설치 방법 (새 프로젝트 적용)

다음 프로젝트를 시작할 때, 최상단 폴더에 생성된 `AI_Agent_SuperPack.zip` 파일의 압축을 새 프로젝트의 최상위 경로에 푸시면 됩니다. (내부의 `.agent` 폴더가 압축 해제됩니다.)

기본적으로 `.agent` 디렉토리에 모든 워크플로우와 시스템 프롬프트가 들어있으며, 이 폴더를 복사하는 것만으로 이전과 동일한 AI 개발 환경(Claude Forge + bkit)을 즉시 구성할 수 있습니다.

---

## 🚀 슈퍼팩으로 할 수 있는 영역 (총망라)

이 패키지는 단순한 코딩(Code Completion) 보조를 넘어, **소프트웨어 개발의 전 생명주기(SDLC)를 관통하는 AI 자동화 파이프라인**을 제공합니다. 

### 1️⃣ 전 자동화 스마트 개발체계 (Auto Complete)
코드 기획부터 테스트 작성, 코드 리뷰, PR(Pull Request) 생성까지 중간에 끊기지 않고 한 번에 실행합니다.
* **사용처**: 새로운 기능 추가, 버그 수정, 대규모 리팩토링의 전체 사이클을 멈춤 없이 빠른 속도로 완료하고 싶을 때.

### 2️⃣ PDCA 기반 체계적 방법론 (Plan-Do-Check-Act)
`bkit` 방법론을 기반으로 문서 기반의 꼼꼼하고 완벽한 개발을 진행합니다. 팀 개발이나 대형 프로젝트에서 오차를 줄입니다.
* **사용처**: 코드 작성 전, 기획서(Plan)와 아키텍처 설계도(Design)를 확실히 세우고, 개발(Do), 결함분석(Check/Analyze), 개선방향 도출(Act/Iterate)을 엄밀하게 적용할 때.

### 3️⃣ 자동화 훅 기반의 코드 품질 및 보안 관제
코드 작성 후 자동으로 보안 취약점을 감지하고, OWASP Top 10 기반의 검수를 실행하며 테스트 커버리지를 최적화합니다.
* **사용처**: 신규 코드를 프로덕션(실도입)에 배포하기 전, 보안 감사를 자동 시뮬레이션할 때.

### 4️⃣ 멀티-에이전트 오케스트레이션(Agent Orchestration)
각 전문 분야별로 최적화된 여러 에이전트가 동시에 협업합니다.
프론트엔드 에이전트와 백엔드 에이전트, 테스팅 전문 QA 에이전트 팀이 톱니바퀴처럼 맞물려 통합 태스크를 수행합니다.

## ⌨️ 핵심 명령어(Slash Commands) 모음 (명령어 사전)

이 패키지는 AI와의 채팅창에 `/` (슬래시) 명령어나 관련 키워드를 입력하여 즉시 호출/동작합니다.

### 🔥 1. 자동화 & 올인원 (제일 강력한 킬러 명령어)
* `/auto [작업설명]` : 초기 계획부터 -> 코드 개발 -> 테스트 -> 리뷰 -> 커밋까지 원스톱 풀사이클 자동 실행. (예: `/auto 구글 소셜 로그인 기능 추가`)
* `/auto --mode bugfix [설명]` : 버그 원인 파악부터 수정, 테스트, 커밋까지 일괄 처리.
* `/auto --mode refactor [설명]` : 사용하지 않는 코드를 제거하고 구조를 개선한 뒤 버그가 없는지 회귀 테스트까지 자동 수행.
* `/sync-docs` : 현재 로컬 환경의 상태(prompt_plan.md 등 프로젝트 룰 문서)를 최신화하여 에이전트들을 최신 동기화.

### 📝 2. PDCA 방법론 (단계별 정밀 설계/개발)
* `/pdca plan [기능명]` : 개발 전 완벽한 기획 및 요구사항 정의서(PRD) 선언.
* `/pdca design [기능명]` : 아키텍처 및 데이터베이스, 시스템 구조 설계도 도출.
* `/pdca do [기능명]` : 기획/설계도를 참조하여 즉시 실제 코드를 생성 및 이식.
* `/pdca analyze` : 생성된 코드가 초기 목적 기획서와 100% 일치하는지 심층 갭(Gap) 분석.
* `/pdca iterate` : 분석된 문제점을 토대로 자동 수정 사이클(최대 5회 루프) 반복.

### 🛡️ 3. 검증 및 퀄리티 컨트롤 (TDD 및 리뷰)
* `/tdd` : "테스트 주도 개발". 테스트 코드를 먼저 만들게 하고, 이 틀에 맞춰 구현 코드를 작성시킨 뒤 리팩토링 진행.
* `/code-review` : 방금 작성된 코드의 품질과 보안(크리티컬, 하이 레벨 보안)을 즉시 점검.
* `/handoff-verify` : 컨텍스트를 새로 고쳐서 빌드, 테스트, 린트를 전부 돌려보고 깨지는 부분이 없는지 검증.
* `/security-review` : STRIDE 위협 모델링 및 SOC2/GDPR 규정 준수 검토 프로세스 실행.

### 🏗️ 4. 구조 탐색 및 고도화
* `/explore` : 대규모 프로젝트에서 버그 원인이나 아키텍처를 찾기 위해 코드베이스를 심층 분석.
* `/orchestrate` : 프론트/백/QA 등 허브-앤-스포크(Hub-and-spoke) 구조로 병렬 복합 태스크 분장 기능 활성화.

## ⚡ 터미널 단축키/Aliasing(별칭) 설정 가능 여부

일반적으로 에이전트 명령어(`/auto` 등)는 AI 인터페이스 채팅창에서 대화 형태로 입력하는 형식입니다.
하지만 Windows 시스템 터미널 자체에서 매번 긴 명령어나 AI 프롬프트를 치기 번거롭다면, PowerShell `Profile`에 Alias(별칭) 단축키를 설정할 수 있습니다.

**터미널 단축어 설정 방법 (PowerShell 활용):**

1. Windows 환경의 터미널(PowerShell)에서 `notepad $PROFILE` 을 입력합니다. (해당 파일이 없으면 텍스트 문서 새로 생성 창 확인)
2. 가장 하단에 아래와 같은 함수 템플릿을 추가하고 저장합니다. (예: `ai-auto` 단축키 등록)

```powershell
# 개발 자동화 단축 명령어 예시 (AI 어시스턴트 CLI인 경우)
# claude-code 나 gemini-cli 등의 쉘 인터페이스로 인자 전달
function Invoke-AIAuto {
    param([string]$task)
    # 실제 사용하시는 AI CLI 실행 바이너리와 결합 (예: claude)
    Write-Host "AI 에이전트에게 다음 작업을 자동 지시합니다: $task" -ForegroundColor Cyan
    # claude /auto $task   # (이와 비슷한 형태로 응용 결합 가능)
}

# ai-auto 단축키 생성
Set-Alias -Name ai-auto -Value Invoke-AIAuto
```
3. 터미널을 완전히 종료 후 다시 열면, `ai-auto "로그인 기능 버그 수정해줘"` 라고 쳤을 때 자동으로 백그라운드에서 AI가 `auto --mode bugfix`를 시작하도록 단축 환경 구성이 가능합니다. (CLI 도구의 인자 전달 기능 지원 여부에 따라 연결 방식은 조금 다를 수 있습니다.)
